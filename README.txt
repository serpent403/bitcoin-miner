1. Size of the Work Unit: The worker gets a subproblem in the sequence format such as A + a * (worker_count). A is the sequence that server(boss) is working on to mine the bitcoins. “worker_count” is nothing but the Nth worker that requested work from the master.

For mining bitcoins the server generates input strings in a particular sequence. For e.g.: If the sequence is : aaa, aab, ......zzz, aaaa, aaab, ........zzzz, aaaaa, aaaab, ......zzzzz,.... . The server starts working on this sequence and as workers keep spawning the pattern is distributed. So, if the 1st worker requests work, it will be assigned the subproblem sequence "aaa" + "a" * 1 => "aaaa" (aaaa, aaab, . . .), and if the 2nd worker requests work it will be assigned the subproblem sequence "aaa" + "a" * 2 => "aaaaa” (aaaaa, aaaab, . . .), and so on.

The best performance is obtained when the server is mining the sequence starting with 7 a’s (aa.. 7times) and subsequent workers are allotted the sub problems with sequence starting with 8 a’s, 9 a’s and so on. This way by incrementing the sequence pattern for every worker spawned (instead of just sticking to one sequence range), the system ensures that input strings never get exhausted. Thus, yielding more and more bitcoins.
